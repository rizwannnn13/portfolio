How to Protect the Server From Attacks By Detecting the Network Using Snort

Introduction

Snort adalah sebuah perangkat lunak yang berfungsi sebagai Intrusion Detection System (IDS) atau Intrusion Prevention System (IPS).Snort disebut sebagai a paket sniffer yang memonitor lalu lintas jaringan, memeriksa setiap paket dengan cermat untuk mendeteksi bahaya muatan atau anomali yang mencurigakan. Lama menjadi pemimpin di antara alat pencegahan dan deteksi intrusi perusahaan, pengguna dapat mengkompilasi Snort di sebagian besar sistem operasi Linux (OSes) atau Unix. Versi juga tersedia untuk Windows.

Snort Uses

  1. Deteksi Serangan: Snort dapat mendeteksi berbagai jenis serangan keamanan, termasuk serangan jaringan yang berpotensi merusak sistem atau mencuri data.
  2. Pencegahan Serangan: Dengan konfigurasi yang tepat, Snort dapat diatur untuk mencegah serangan sebelum mereka berhasil merusak atau merusak server.
  3. Monitoring Jaringan: Snort memberikan kemampuan untuk memantau lalu lintas jaringan secara real-time, sehingga administrator dapat melihat aktivitas yang mencurigakan atau tidak diinginkan.
  4. Pengumpulan Informasi: Snort menyediakan laporan dan informasi tentang serangan yang berhasil dideteksi, yang memungkinkan administrator untuk melakukan analisis dan menanggapi serangan tersebut.
  5. Perlindungan Data: Dengan mendeteksi dan mencegah serangan, Snort membantu melindungi data sensitif yang disimpan atau diproses oleh server.
  6. Penyelidikan Keamanan: Snort dapat digunakan sebagai alat untuk menyelidiki insiden keamanan yang terjadi di jaringan atau server, membantu administrator untuk memahami sumber serangan dan langkah-langkah yang diperlukan untuk memperbaikinya.
  7. Integrasi dengan Sistem Keamanan Lain: Snort dapat diintegrasikan dengan perangkat lunak keamanan lainnya, seperti firewall atau sistem manajemen keamanan lainnya, untuk meningkatkan lapisan pertahanan keamanan keseluruhan.

Toppologi

Configuration

Debian 10- Router

Install Package

apt install snort

Change File Snort

nano /etc/snort/snort.conf

ipvar HOME_NET
*menjadi 
ipvar HOME_NET 192.168.91.0/24

nano /etc/snort/rules/local.rules 

alert icmp any any -> any any (msg:"PING DETECTED!";sid:10000001;rev:0)
alert tcp any any -> any 161 (msg:"NMAP SCAN DETECTED";sid:10000002;rev:1)
alert tcp any any -> any 22 (msg:"SSH login DETECTED";sid:10000003;rev:1)

Restart Snort Service

/etc/init.d/snort restart

Run Snort

snort -A console -q -c /etc/snort/snort.conf -i ens33

KaliLinux

nmap -sN (IP Debian10-Router)
nmap -sN 192.168.91.150

Pastikan di Debian 10- Router terdapat pemberitahuan jika ada yang sedang melakukan port scanning.
