Penjelasan

SFTP adalah singkatan dari SSH File Transfer Protocol. Fungsi utamanya yakni membuat aktivitas pentransferan data antara client dan server menjadi lebih aman. Dalam artian lain, metode ini bisa mencegah terjadinya peretasan atau pembajakan oleh hacker. Namun, jika dilihat dari kepanjangannya, apakah SFTP sama dengan FTP? Serupa tapi tak sama, mungkin itu adalah jawaban paling tepat. Terdapat perbedaan FTP dan SFTP yang cukup signifikan, terlebih dari segi keamanannya.

Perbedaan

  1. FTP adalah protokol jaringan yang digunakan untuk bertukar file melalui jaringan TCP/IP. Sementara itu, SFTP adalah protokol jaringan yang memungkinkan akses file, transfer, dan manajemen melalui aliran data.
  2. FTP tidak menyediakan saluran keamanan apa pun untuk mentransfer file antara host. Namun, protokol SFTP memilikinya.
  3. FTP dapat diakses secara anonim, dan dalam banyak kasus tidak dienkripsi. Kebalikannya, SFTP adalah protokol yang selalu mengenkripsi data.
  4. Di satu sisi, FTP membuat koneksi menggunakan koneksi kontrol pada port TCP 21. Di sisi lain, SFTP adalah protokol yang mentransfer file di bawah koneksi aman milik protokol SSH antara klien dan server.

Konfigurasi

Server

Interfaces

ens33 LAN (Server): 20.1.1.20/24

Package

  • SSH
  • Apache2
  • Proftpd (New)

Install Package

apt install proftpd

Konfigurasi Proftpd

nano /etc/proftpd/proftpd.conf

#DefautRoot
Diubah menjadi
DefautRoot

Setelah itu kita restart service proftpd agar bisa berjalan dengan lancar

systemctl restart proftpd

Membuat group

groupadd sftp-rizwan

Membuat User

Buat user didalam group yang akan digunakan sebagai user SFTP

useradd -G sftp-rizwan --shell /bin/false itsupport
useradd -G sftp-rizwan --shell /bin/false sysadmin
useradd -G sftp-rizwan --shell /bin/false developer

Membuat Directory SFTP

mkdir -p /srv/sftp-office

Membuat Directory Untuk User

Hal yang perlu kita lakukan terlebih dahulu adalah masuk ke directory SFTP yang telah kita buat tadi dengan cara:
cd /srv/sftp-office

mkdir -p file-itsupport
mkdir -p file-sysadmin
mkdir -p file-developer

Mengganti Password User

passwd itsupport
passwd sysadmin
passwd developer

Konfigurasi SSH

(Tambahkan konfigurasi SSH dipaling bawah)
nano /etc/ssh/sshd_config

Match Group sftp-rizwan
    PasswordAuthentication yes
    PermitTunnel no
    AllowAgentForwarding no
    AllowTcpForwarding no
    X11Forwarding no
    ChrootDirectory /srv/sftp-office
    ForceCommand internal-sftp

Pengujian

20.1.1.20 itsuppport/sysadmin/developer (password) port:22

Buka Filezilla di Windows dan usernya bisa gunakan itsupport, atau sysadmin, atau developer dengan password yang telah kalian ganti
